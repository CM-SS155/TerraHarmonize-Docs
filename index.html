def right_choice(self) ‑> list

Applicabe to states like MP, MH, OD, and RJ if survey number does not have regional/english alphabet.

ⓘ  Note

Here the survey number is split based on '/'. But if you find '-' or '\' as a split, go for right_choice_AP. Basically all states that do not have regional language issues and the client data format matches the DB format of survey number can go with this method.

def right_choice_AP(self, include: Literal['normal', 'both'] = 'normal', state: Optional[Literal['TS', 'AP', 'MH', 'KA', 'RJ', 'OD']] = None, split_pat: str = '\\/|\\-')

Used for matching AP, Telangana, and all other states that have regional/english alphabet in survey_no. This can be updated to any new state, but a transliteration has to be added. This can also be used if the survey_number has different pattern for spliting.

  • include: It includes normal (default) which is used when you want to match without changing the characters i.e the english alphabet will not converted to Telugu and both which is first matched without changing alphabets and then matched by changing alphabets example: 14/1A will first matched as 14/1A and then converted to 14/1అ and matched

  • state: The state you are working with. By default state is None, and if state is None then 'include' should be normal. For 'include' as normal, 'state' can be anything. But, if 'include' is 'both', then state should be either TS, AP, MH, RJ, OD or KA

  • split_pat: The pattern (regex prefered) based on which the text has to split. The default is \/|- i.e. split is based on either '/' or '-'. example: 42-2/1 will become [42,2,1]
